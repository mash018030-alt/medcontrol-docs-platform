# MedControl — контент документации (архив)

> **Репозиторий архивный.** С **2026** единый **источник правды** для текстов, изображений и новостей — репозиторий платформы **[medcontrol-docs-platform](https://github.com/mash018030-alt/medcontrol-docs-platform)** (каталог **`content/`** в корне). Движок подключает его как **`engine/public/content/`** (связь создаётся скриптом **`engine/scripts/ensure-content-link.cjs`**).

Не используйте этот репозиторий для текущих правок документации. Снимок здесь сохранён для истории, старых ссылок и сравнения.

---

## Куда смотреть сейчас

| Что нужно | Где |
|-----------|-----|
| Редактировать статьи и медиа | [**medcontrol-docs-platform**](https://github.com/mash018030-alt/medcontrol-docs-platform) → **`content/`** |
| Код сайта, CI, PDF | Тот же репозиторий → **`engine/`** |
| Шпаргалка по дереву репо | В платформе: **`engine/repo-docs/REPO-LAYOUT.md`** |
| Базовая линия ссылок «до объединения» | В платформе: **`engine/archive/migration/`** |

## Было в этом репозитории (справка)

Раньше контент вели отдельно и подключали к движку через submodule в `docs-app/public/content` или копирование в CI. После объединения submodule удалён, контент лежит в **`content/`** внутри **medcontrol-docs-platform**.

Целевое дерево и словарь имён по-прежнему описаны в **`content/repo_docs/REPO_STRUCTURE.md`** (в репозитории платформы).

| Раздел (исторический) | Назначение |
|----------------------|------------|
| `0_docs/` | Дерево пользовательской документации MedControl |
| `1_news/` | Новости |
| `images/` | Медиа к статьям, дашборды, логотип |
| `references/` | Эталоны PDF, требования, вспомогательные материалы |
| `repo_docs/` | Заметки про устройство контент-репозитория (актуальная копия логики — в **`content/repo_docs/`** на платформе) |
