# Бренд платформы документации (репозиторий движка)

Здесь — то, что описывает **организацию самой платформы документации** относительно бренда: прежде всего цвета и согласование UI (человекочитаемый референс). **Логотипы, иллюстрации к статьям, PDF-эталоны и прочий контент** лежат в **`content/`** (корень репозитория), а не в этой папке.

Материалы в `engine/docs-brand/` держим **вне** публикуемых статей, чтобы они не попадали в поиск по документации и не смешивались с материалами для читателей.

- **`colors/`** — палитра в Markdown для согласования с дизайном.
- **Отображение в UI:** переменные и шрифты продукта задаются в **`content/brand/theme.css`** (подключается движком после `App.css`; см. **`content/brand/README.md`**). Нейтральные дефолты платформы — в **`engine/src/App.css`** (`:root`).
