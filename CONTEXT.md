# Контекст проекта MedControl Docs

**Для новых чатов:** напиши ассистенту: «используй материалы из **docs-app/public/content/references/** (**references/README.md**), палитру из **`design/colors/`** в корне репозитория, brief.md и при необходимости CONTEXT.md».

---

## Проект

Прототип платформы пользовательской документации для **MedControl** (демо, локально, без пуша в GitLab).

Порядок работ: сначала навайбкодить платформу, потом перенести PDF / Google Docs в Markdown.

---

## Организация

- **Рабочая папка** — отдельная на компе (не внутри git-learning).
- **references/** — в **репозитории контента** (`docs-app/public/content/references/` после submodule): тексты требований и референс по видео, экспорты руководств (HTML + `images/`), лого и прочие медиа. См. **`references/README.md`** там же. Скрипт **`docs-app/scripts/html-to-md.cjs`** читает HTML из `public/content/references/manuals/...`.
- **design/colors/** — в **репозитории движка** (корень, рядом с `docs-app/`): Markdown-референс цветовой палитры и при необходимости образцы `*.png`. Не входит в `public/content/`, чтобы не смешиваться со статьями и поиском по ним. Начало — **`design/colors/color-palette.md`**.
- **brief.md** — описание платформы и требований (при необходимости).
- Несколько репозиториев на компе — нормально; ассистент видит только папку, открытую в Cursor (File → Open Folder).
- Текст из чата ассистент может сохранять в файлы.

## План

| Этап | Содержание |

## Кнопки подтверждения агента (только этот репо)

Чтобы не жать вручную при работе в чате, всё ниже можно ослабить **только для этой папки** через **Cursor Settings** (Ctrl+Shift+J) → **Agents** → **Auto-Run** и **Protection**; часть настроек продублирована в `.vscode/settings.json`.

| Что видишь в чате | Что это | Где отключать «спрашивать каждый раз» |
|-------------------|--------|----------------------------------------|
| **Run** | Запуск команды в терминале | Auto-Run → «Run in Sandbox» или «Run Everything» |
| **Fetch** | Запрос по URL (достать содержимое страницы) | Входит в общий Auto-Run; при «Run in Sandbox» может всё равно спрашивать сетевые запросы — тогда только «Run Everything» или добавление в allowlist |
| **Continue** | Агент ждёт твоего «продолжай» (вопрос или пауза) | Это не кнопка безопасности, а режим «спрашивать перед продолжением»; в настройках чата/агента ищи опции вроде «Ask before continuing» / «Pause for approval» и выключи |
| **MCP** | Вызов внешнего инструмента (MCP) | Auto-Run → Protection → **MCP Allowlist** (добавить нужные инструменты) или «Run Everything» |
| **Browser** | Действия в браузере (скриншот, переход) | Protection → **Browser Protection** выключить = разрешить авто-запуск браузера |
| **Редактирование .vscode / .cursor** | Правка конфигов воркспейса | По умолчанию всегда с подтверждением; в Protection нет отдельного выключателя — только «Run Everything» |
| **Удаление файлов** | Удаление файлов агентом | Protection → **File-Deletion Protection** выключить = разрешить удалять без спроса |
| **Dotfile / внешние файлы** | Правка .gitignore и т.п., файлы вне папки | Protection → **Dotfile Protection**, **External-File Protection** выключить при необходимости |

Итого по типам: **Run** и **Fetch** — терминал и сеть (Auto-Run). **Continue** — пауза/вопрос агента (настройки чата). **MCP / Browser / удаление / dotfile** — отдельные переключатели в **Protection**. Для «всё разрешить только здесь» попробуй в этой папке задать в `.vscode/settings.json` режим и снять защиты (см. файл).

## Auto-Run в этом проекте

В `.vscode/settings.json` для этой папки включён режим **Run in Sandbox** и ослаблены защиты, чтобы реже нажимать Run/Fetch/Continue и др. Если не сработает — то же самое вручную: **Cursor Settings** (Ctrl+Shift+J) → **Agents** → **Auto-Run** и **Protection** (глобально или «Save as Workspace», если есть).
