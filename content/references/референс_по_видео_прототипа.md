# Референс по видео прототипа платформы

Этот файл описывает интерфейс, показанный на видео с записью экрана.  
Используй его как референс для разработки будущей платформы документации.

## Что показано на видео

На видео показан прототип платформы документации в формате веб-интерфейса.

Интерфейс похож на документационный портал:  
есть навигация по разделам, область с контентом статьи и общая структура, удобная для чтения инструкций.

## Основные элементы интерфейса

### 1. Левое меню навигации
В интерфейсе есть боковое меню слева.

Оно используется для:
- перехода между разделами документации
- показа структуры материалов
- доступа к вложенным страницам

Левое меню должно быть основным способом навигации по документации.

### 2. Верхняя часть страницы
В верхней части интерфейса есть шапка страницы.

В ней могут располагаться:
- название продукта / платформы
- строка поиска
- элементы глобальной навигации

### 3. Основная область контента
В центре экрана находится содержимое страницы документации.

Там отображаются:
- заголовок статьи
- текст инструкции
- подразделы
- списки
- таблицы или изображения, если они нужны

Это главная зона чтения документации.

### 4. Структура документационной страницы
Страница выглядит как полноценная статья базы знаний или пользовательской документации.

Типичная структура:
- заголовок страницы
- краткое описание
- блоки с подзаголовками
- пошаговые инструкции
- дополнительные материалы

### 5. Навигация внутри документации
На видео видно, что пользователь перемещается между разделами платформы.

Это значит, что будущая платформа должна поддерживать:
- переход между страницами
- логичную иерархию разделов
- понятную структуру документации

## Как использовать это как референс

При разработке платформы ориентируйся на следующие принципы из видео:

- документация должна выглядеть как сайт, а не как набор файлов
- у платформы должна быть боковая навигация
- основное содержимое должно открываться в центральной области
- структура должна быть удобной для чтения длинных инструкций
- интерфейс должен быть простым, понятным и похожим на современные documentation portals

## Что нужно воспроизвести в будущем прототипе

Нужно использовать показанный на видео паттерн интерфейса:

- documentation-style layout
- sidebar navigation
- article content area
- hierarchical structure of pages
- удобный просмотр и чтение документации

Не нужно пытаться повторять видео буквально пиксель в пиксель.  
Нужно использовать его как референс по общей структуре, логике интерфейса и навигации.
