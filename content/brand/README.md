# Тема оформления (контент)

Здесь лежит **оформление оболочки** сайта документации для конкретного продукта: цвета, шрифты, градиенты. Это **не** статьи для читателей — папка не должна попадать в навигацию и поиск (см. скрипты проверки ссылок в `engine`).

## Как посмотреть «пустую» платформу (нейтрал, без лого MedControl)

В каталоге **`engine`** в файле **`.env.local`** (в git не коммитится) временно задайте и перезапустите `npm run dev`:

```env
VITE_DOCS_USE_CONTENT_THEME=0
VITE_DOCS_LOGO_URL=
```

**Важно:** при `VITE_DOCS_USE_CONTENT_THEME=0` нейтральные цвета из **`App.css`** действуют на **весь интерфейс движка**: шапка, вкладки «Документация / Новости», поле поиска, **карточки на главной**, боковое меню, кнопки и т.д. Это не «только рамка»: продуктовая палитра из **`theme.css`** целиком отключена.

Тексты и заголовки на страницах при этом **как в вашем `content/`** (например, упоминания MedControl в описаниях разделов) — это содержимое статей и подписей, а не тема. Получается предпросмотр «универсальной оболочки» поверх **того же** контента.

Чтобы снова видеть оформление MedControl (синие акценты, карточки, лого по умолчанию) — **удалите эти строки** или закомментируйте их в `.env.local` и перезапустите dev.

Полностью пустой сайт без статей локально не поднимается: нужна связка **`public/content`** из `ensure-content-link`.

## Файлы

| Файл | Назначение |
|------|------------|
| **`theme.css`** | CSS‑переменные `:root` (акценты, фоны навигации, баннер новостей, макросы «Примечание»/«Внимание», тень баннера) и подключение шрифтов. Подхватывается движком по URL `/content/brand/theme.css` после `App.css`. |

**Логотип в шапке** — `VITE_DOCS_LOGO_URL` (см. `engine/.env.example`): не задана — лого MedControl по умолчанию; пустая строка — без логотипа.

**Заголовок вкладки** (`<title>`) — **`VITE_DOCS_SITE_TITLE`** в `engine` (`vite.config.js`); если не задана — «MedControl — Документация».

**Favicon** — **`VITE_DOCS_FAVICON_URL`** и при необходимости **`VITE_DOCS_FAVICON_TYPE`** (там же). Путь вида `content/brand/favicon.svg` ведёт в каталог контента; по умолчанию остаётся `engine/public/icons/favicon.svg`.

**Подпись ссылки с логотипом** (screen readers) — **`VITE_DOCS_HOME_ARIA_LABEL`** в клиентской сборке (`resolveDocsHomeAriaLabel.js`).

Другая команда может **заменить** `theme.css` (и при необходимости добавить `fonts/` с `@font-face`) под свой бренд, не трогая ядро движка.

Текстовый референс палитры (таблицы HEX, согласование с дизайном) по-прежнему можно вести в **`engine/docs-brand/colors/`** или дублировать кратко здесь — источник правды для **отображения в браузере** — переменные в `theme.css`.
