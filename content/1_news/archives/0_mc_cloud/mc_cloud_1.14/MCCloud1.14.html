MEDCONTROL CLOUD

ЧТО НОВОГО В ВЕРСИИ 1.14

MC Cloud 1.14

В этом обновлении основной фокус был на улучшении работы с отчётами. Мы сделали более дружелюбные формы создания отчётов и добавили возможность указания временных зон.

Кроме этого, появился долгожданный фильтр осмотров по медицинской организации, и ещё несколько доработок.

Новые возможности

Отчёты

Обновлённая форма создания отчётов

Чтобы проще формировать отчёты, обновили интерфейс, подсказки и надписи в разделе.

  • Если не заполнено обязательное поле,  система подскажет, что нужно заполнить.

  • При ошибках с подписанием отчёта через ЭЦП введённые данные также не теряются – можно сформировать отчёт без повторного заполнения.
  • В «Медицинском журнале» по умолчанию указан самый распространенный тип журнала – «Предрейсовый, предсменный».
  • Удобный выбор времени и даты. Вместо двух отдельных полей «С» и «ПО» при выборе периода отчёта сделали выпадающий календарь, где можно указать сразу диапазон дат (максимальный период – 31 день). Также добавили поле «Выбор часового пояса», по которому будут отображаться события в отчёте.

Разделение биллингового отчёта

Биллинговый отчёт можно сформировать отдельно по контрагенту и отдельно – по медорганизации. Благодаря этому у организаций не возникает конфликта часовых поясов, по которым отображаются события в отчёте.

Упрощённая выгрузка меджурнала по контрагенту

При формировании Медицинского журнала убрали поле выбора работника на вкладке «Контрагент». Раньше в журнале могла быть заполнена только одна вкладка с выбранным сотрудником организации, а вкладки с остальными организациями оставались пустыми.

Теперь меджурнал по контрагенту можно сформировать без лишних данных.

Организации

Расширенные фильтры

В раздел «Организации» добавили кнопку для фильтрации списка.

Организации можно отфильтровать по:

  • типу – Компания/ИП/Группа/Подразделение;
  • статусу – Активна/Неактивна/Деактивирована;
  • наличию точки выпуска – поиск организаций с возможностью установки стационарных ПАК;
  • наличию медицинского профиля – медицинская организация или нет;
  • наличию признака «контрагент» – организация может платить за осмотры;
  • обслуживающей медорганизации (обрабатывающей осмотры);
  • организации-контрагенту (оплачивающей осмотры).

Список назначенных организаций

В карточке медорганизации появилась кнопка «Назначенные организации» – по ней можно посмотреть, чьи осмотры эта медорганизация обрабатывает. Раньше такие данные можно было увидеть только в блоке «Профиль медобслуживания» в карточке организации, к которому не у всех был доступ.

Осмотры

Вместе с Организациями расширили фильтрацию в разделе «Осмотры».

Фильтр по медоператору

Появился долгожданный фильтр для поиска осмотров по медорганизации, очень актуальный для пользователей с широким доступом.

Поиск по дочерним

Раньше при поиске осмотров по организации мы не получали осмотры, принадлежащие её подразделениям и дочерним компаниям. Теперь этим можно управлять. Для поиска по дочерним организациям появился переключатель.

ПАК

Вкладка «Журналы»

Чтобы упростить навигацию и сделать поиск лог-файлов более очевидным, вкладку «Отчёты» в карточке ПАК переименовали в «Журналы».

Детализация статусов при запросе логов

Доработали отображение статусов обработки логов ПАК: теперь видно, кто запросил файл, когда, за какой период, и много другой полезной информации. Если возникает ошибка, показываем точную причину: ПАК не отвечает/нет логов за период/не удалось загрузить файл.

Также название файла стало более информативным - в нём теперь содержится заводской номер ПАК и временной интервал.

Исправление ошибок

Устранили недочёты с отсутствием данных или их неактуальностью, а также с отображением данных пользователям без доступа:

Старые данные после перепривязки ПАК
После создания нового ПАК в статистистике могли отображаться старые данные. Исправили – теперь показываем актуальную информацию о новом ПАК.

Импорт работников из шаблона
При использовании стандартного шаблона, импорт работников завершался с ошибкой «Превышено время обработки файла» – починили.

Ошибка поиска пользователя с некорректным табельным номером
Ранее нескольких пользователей с некорректным табельным номером нельзя было найти в поиске – система выдавала непонятную ошибку. Поправили – нужного пользователя можно найти и открыть его карточку.

Отображение вкладки «Уведомления» пользователям без доступа
Пользователи без доступа к управлению подписками видели вкладку «Уведомления», но с ошибкой «Недостаточно прав, чтобы смотреть подписки». Сейчас вкладка скрыта для таких пользователей.

Отображение событий в разделе «Аудит» пользователям без доступа
Ранее в «Аудите» пользователи могли видеть события организаций, к которым не имели доступа. Настройки изменили – теперь такие события видят только пользователи с доступом.

Неактуальное название ПАК в «Аудите»
После привязки ПАК к организации в разделе «Аудит» отображалось старое, временное имя ПАК до привязки. Сейчас показываем заводской номер ПАК, а при его отсутствии – корректное имя.

Отсутствие перевода статуса «Сбой оборудования»
В карточке и списке осмотров статус «ConnectionFailed» отображался без перевода на русский. Теперь показывается, как «Сбой оборудования».

Старый статус просроченного документа

Было: в карточке пользователя входящие документы оставались в статусе «На рассмотрении» даже после окончания срока их действия.

Стало: статус просроченного документа меняется на «Истек срок действия» автоматически.

Неверное отображение автора документа

Было: после загрузки входящего документа в поле «Добавил» отображалось неверное имя пользователя.

Стало: отображается верное ФИО пользователя, добавившего документ.

Нет ФИО в архивированном фото
В карточке осмотра с архивированным фото не отображался пользователь, который его архивировал. Исправили – показываем ФИО.

Неверное время подписи решения в осмотре
Если осмотр несколько раз возвращался медработнику из-за сбоев подписания ИДС[1], в статистику ЕРП[2] могло попасть неверное время подписания – починили.

Ошибка при завершении смены медработника
Если медработник превышал планируемое время смены и выходил из линии, появлялась ошибка «Смена не найдена по запросу». Проблему исправили, смена завершается корректно.

Неверное сообщение при восстановлении интернета
У медработников в линии при коротком разрыве соединения появлялась ошибка «Возникла небольшая проблема», хотя работа продолжалась корректно. Восстановление соединения улучшили – больше система не показывает лишних ошибок.

[1] Информированное добровольное согласие.

[2] Единый реестр перевозчиков.
