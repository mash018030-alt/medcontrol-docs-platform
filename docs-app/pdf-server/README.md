# PDF-сервис (Playwright)

Генерирует PDF той же вёрсткой, что в Chromium (печать страницы), в отличие от html2canvas в браузере.

## Запуск локально

1. Установка (из каталога `pdf-server`):

```bash
npm install
```

2. Поднять сервис:

```bash
npm start
```

По умолчанию порт **3001**. Разрешённые хосты страниц по умолчанию: `localhost:5174`, `127.0.0.1:5174` (порт Vite из `vite.config.js`).

3. Запустите фронт: `npm run dev` в `docs-app` (порт **5174**). В режиме разработки Vite **проксирует** `POST /api/pdf` на `http://127.0.0.1:3001`, поэтому **`VITE_PDF_SERVICE_URL` в `.env.local` не обязателен** — достаточно поднятый `pdf-server`.

4. (Опционально) В корне `docs-app` можно задать явный URL сервиса — тогда запрос пойдёт напрямую, минуя прокси:

```env
VITE_PDF_SERVICE_URL=http://127.0.0.1:3001
```

Полезно, если прокси мешает или сервис на другом хосте. После смены `.env` перезапустите `npm run dev`.

5. Проверка: в браузере откройте `http://127.0.0.1:3001/health` — ответ `{"ok":true}`. Если кнопка PDF всё равно падает с «Failed to fetch» — сервер не запущен или порт занят.

Если `pdf-server` недоступен, для **одной статьи** скачивается **растровый** PDF (html2pdf); качество переносов хуже, чем у Chromium.

## Продакшен

- Разверните `pdf-server` на том же VPC/хосте или рядом (Node + зависимости Playwright тяжелее обычного API).
- Сервис открывает в Chromium **тот же URL**, что у пользователя, с параметром `mc_pdf=1` (без шапки и бокового меню). Хост страницы и хост API могут различаться; главное, чтобы с **машины, где крутится pdf-server**, этот URL был **доступен по сети** (localhost только если фронт и API на одной машине).
- Задайте `ALLOWED_HOSTS` списком хостов **вашего** сайта документации (через запятую), чтобы исключить SSRF.
- Задайте `CORS_ORIGIN` на точный Origin фронта, не `*`, если возможно.
- В прод-сборке фронта укажите `VITE_PDF_SERVICE_URL` на публичный URL сервиса (или прокси с того же домена, например `/api/pdf`).

## Переменные окружения

| Переменная                  | Описание |
|-----------------------------|----------|
| `PORT`                      | Порт сервера (по умолчанию 3001) |
| `ALLOWED_HOSTS`             | Разрешённые `host:port` для поля `url` в запросе |
| `CORS_ORIGIN`               | Заголовок `Access-Control-Allow-Origin` |
| `PDF_SINGLE_PAGE_MAX_PX`    | Порог высоты `.docs-markdown-root` в px: если контент не выше — PDF **одной длинной страницей** (удобно для FAQ без разрывов заголовок/таблица). По умолчанию `24000`. Длиннее — многостраничный A4 + аккуратные переносы в вёрстке `mc_pdf`. |
