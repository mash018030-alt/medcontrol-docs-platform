# Платформа документации MedControl (прототип)

Локальный прототип на React + Vite. Шапка, левое меню разделов, статья по центру, оглавление справа, навигация «Предыдущая / Следующая».

## Контент (git submodule)

Тексты и медиа живут в отдельном репозитории **`medcontrol-docs-content`**. В этом проекте они подключены как submodule в каталог **`public/content/`**.

После клона движка:

```bash
git submodule update --init --recursive
```

Если submodule ещё не создан на GitHub: создайте репозиторий `medcontrol-docs-content`, запушьте туда контент, убедитесь, что URL в **`.gitmodules`** совпадает с ним, затем выполните команды выше.

Локально submodule изначально мог быть привязан через `file://` — после смены URL на HTTPS выполните **`git submodule sync`**.

## Запуск

Установка зависимостей (один раз):

```bash
cd docs-app
npm install
```

Запуск в режиме разработки:

```bash
npm run dev
```

Открой в браузере адрес, который покажет терминал (обычно **http://localhost:5173**).

## Что есть сейчас

- **Шапка** — логотип MedControl, заглушка под поиск
- **Левое меню** — дерево разделов (Начало работы, Раздел 1, Раздел 2), переход по статьям
- **Статья** — загрузка Markdown из `public/content/`, рендер заголовков, списков, таблиц, кода
- **Оглавление справа** — якоря по H2/H3 внутри статьи
- **Предыдущая / Следующая статья** — внизу страницы

## Статьи и меню

Markdown лежит в **`public/content/`** (submodule). Структура левого меню и порядок «Предыдущая / Следующая» задаются в **`src/data/nav.js`** — пути там должны совпадать с путями к `.md` внутри submodule.

**Маркированные списки** в контенте оформляй через **дефис** (`- `), не через звёздочку — так проще читать рядом с жирным `**…**` и совпадает с типичным стилем GFM/Prettier.

**Карусель** (`.docs-carousel` в markdown): стрелки «назад» / «вперёд» должны иметь классы **`docs-carousel-arrow-prev`** и **`docs-carousel-arrow-next`** внутри **`docs-carousel-nav`**. Тогда после загрузки статьи скрипт в **`Article.jsx`** сам скрывает лишнюю стрелку на первом и последнем слайде и не зацикливает переключение. Для активной точки по-прежнему нужны CSS-правила вида  
`.docs-carousel [id="…-1"]:checked ~ .docs-carousel-nav .docs-carousel-dots label[for="…-1"]` под конкретные `id` радиокнопок (или расширить общий стиль при необходимости).

## Сборка статики

```bash
npm run build
```

Результат в папке `dist/` — можно выложить на любой хостинг статики.
