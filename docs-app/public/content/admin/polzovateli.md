# Пользователи

## Доступы пользователей

При создании пользователя в MedControl выбирается его роль – работник, менеджер, администратор и т.п. У каждой роли есть определенная группа разрешений на совершение действий в АРМ[^1]. Например, администратор может создавать новые организации, но не может просматривать медицинскую информацию, а начальник охраны труда, наоборот – может просматривать карточку осмотра с медицинской информацией, но без доступа к созданию организаций.

Ниже – информация о том, какие группы разрешений доступны пользователям. Расшифровка групп (какие действия в АРМ с каждой группой можно совершать) вынесена в отдельную статью: [Группы разрешений пользователей](/admin/gruppy-razresheniy-polzovateley). Группы разрешений, недоступные ни одному пользователю определенного типа, в таблицу не включены. Пример: пользователям типа «Менеджеры» недоступна ни одна группа разрешений, связанная с осмотрами – это доступы в большей части для «Медработников». Следовательно, эта группа в таблице с «Менеджерами» не фигурирует.

**Условные обозначения:**

- ✅ – группа доступна
- ❌ – группа недоступна
- ☑️ – фактически группа недоступна, но её отдельные разрешения (действия) включены в более обширную доступную пользователю группу. Пример: у Менеджера есть группа «Орг.структура: управление», но нет «Орг.структура: просмотр», т.к. управление без просмотра недоступно.

[^1]: Исключая пользователей с ролью «Работник» – у них доступа к АРМ нет.

### Работники

<details class="docs-disclosure">
<summary>Доступы</summary>
<div class="docs-disclosure-content">
<p>Пользователям с ролью «Работник» (тип «Работники») доступны только три группы разрешений:</p>
<ul>
<li>просмотр своей медицинской информации;</li>
<li>отмена осмотра;</li>
<li>управление своей привязкой Telegram.</li>
</ul>
</div>
</details>
