# Частые вопросы

| Вопрос | Ответ |
|--------|-------|
| **Организации** | |
| Как создать дочернюю организацию? | Создать организацию двумя способами: через раздел «Организации» (кнопка «Добавить») или из карточки существующей головной организации (кнопка «Добавить» – «Организацию»). Подробнее – [Создание дочерней организации](/0_docs/2_admin/articles/02_organizacii_00#создание-дочерней-организации). |
| Что такое платёжный профиль и как его настроить? | Платёжный профиль определяет, какое юридическое лицо оплачивает осмотры организации. По умолчанию у дочерней организации наследуется платёжный профиль головной организации, при необходимости его можно изменить при создании организации или добавить позже в настройках. Подробнее – [Создание дочерней организации](/0_docs/2_admin/articles/02_organizacii_00#создание-дочерней-организации) и [Редактирование](/0_docs/2_admin/articles/02_organizacii_01_deystviya_s_organizaciej#редактирование). |
| **Осмотры** | |
| Какие виды осмотров доступны для прохождения? | По умолчанию доступны предрейсовые, послерейсовые и медосмотры в течение рабочей смены. Дополнительно можно добавить предсменные, послесменные осмотры и алкотестирование. Подробнее – [Виды осмотров](/0_docs/2_admin/articles/02_organizacii_02_nastroyki_organizacii#виды-осмотров). |
| Как отменить необработанный осмотр? | При наличии соответствующего доступа необработанный осмотр можно отменить в карточке работника в блоке «Текущий осмотр» по кнопке «Отменить». Подробнее – [Ручные операции над осмотрами](/0_docs/2_admin/articles/07_osmotry#ручные-операции-над-осмотрами). |
| **ПАК** | |
| Как зарегистрировать новый ПАК? | Новый ПАК можно добавить через раздел «ПАК» (кнопка «Добавить») либо из карточки организации («Добавить» – «Добавить ПАК»). Подробнее – [Создание ПАК](/0_docs/2_admin/articles/08_pak#создание-пак). |
| **Админпанель** | |
| Что такое Админпанель? | В «Админпанели» собраны инструменты для контроля работы системы: журнал аудита изменений, техническая информация по пользователям, организациям и ПАК. Через Админпанель можно просматривать все действия, выполненные в системе, фильтровать события и проверять детали изменений. Исключая пользователей с ролью «Работник» — у них доступа к АРМ нет. |
