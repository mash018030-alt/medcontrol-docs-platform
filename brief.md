# Бриф: платформа документации MedControl

Краткое описание **движка** пользовательской документации (не путать со статьями для конечных пользователей MedControl — они в отдельном репозитории контента).

## Зачем это

Нужна собственная веб-платформа: навигация по разделам, поиск по тексту, чтение статей в Markdown, оглавление статьи, переходы «назад / вперёд», экспорт в PDF там, где это предусмотрено. Контент (тексты, скриншоты, лого, PDF-эталоны для сверки) живёт **в submodule** `docs-app/public/content/`. Оформление относительно бренда (палитра и т.п.) согласуется по материалам в **`design/`** в этом репозитории.

## Границы и приоритеты

- Платформа ориентирована на **читаемость и предсказуемую навигацию**, а не на экспериментальный UI.
- Детальные текстовые требования к функциям при необходимости дополняются в **`docs-app/public/content/references/`** (репозиторий контента) и в **`CONTEXT.md`** здесь — там же порядок репозиториев, этапы и служебные заметки для работы в Cursor.
