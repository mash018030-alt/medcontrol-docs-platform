# Устройство репозитория: движок vs контент

Краткая шпаргалка, чтобы не путать **код платформы**, **регламенты про движок** и **тексты статей MedControl**.

| Путь | Что это |
|------|---------|
| **`content/`** | Корневой каталог **пользовательского контента**: `.md` (`0_docs/…`), картинки, `references/`, новости. **Источник правды** для текстов сайта. |
| **`engine/`** | Веб-приложение (React + Vite): UI, роутинг, поиск, сборка. Это **движок**, не папка со статьями для читателя. |
| **`engine/public/content/`** | Связь (junction/symlink) на **`../content`**, создаётся **`npm install`** / **`ensure-content-link.cjs`**. Нужна Vite для URL `/content/…`. В git не хранится (см. **`engine/.gitignore`**). |
| **`engine/engine-docs/`** | Служебные **markdown-заметки только про движок** (PDF, таблицы, деплой). **Не** попадают на сайт и **не** в поиск по статьям. Имя папки намеренно не `docs`, чтобы не смешивать с материалами в **`content/`**. |
| **`engine/docs-brand/`** | Референс бренда UI платформы доков (палитра и т.п.), **внутри движка** — не про статьи и не «дизайн продукта MedControl». |

Индекс файлов в `engine-docs/`: **[`engine/engine-docs/README.md`](engine/engine-docs/README.md)**.

**Канон:** на верхнем уровне репозитория — **`content/`** и **`engine/`**; палитра и прочий бренд движка — **`engine/docs-brand/`**.

---

## Другие имена, которые иногда путают

| Имя | Почему может сбивать | Переименовывать? |
|-----|----------------------|------------------|
| **`content/0_docs/`** | Слово **docs** рядом с «документацией про движок»; префикс **`0_`** выглядит как служебная нумерация. По смыслу это **дерево разделов пользовательской документации MedControl** (URL, `nav.js`). | **Почти никогда:** смена затронет все пути статей, меню, редиректы, внешние закладки. Имеет смысл только как большая миграция с ТЗ. |
| **`content/references/`** | «References» ассоциируется со списком литературы; здесь **эталоны, PDF, экспорты руководств** для сверки. | Переименование = массовая правка путей в контенте. Проще опираться на **`references/README.md`**. |
| **`engine/docs-brand/`** | **Бренд / палитра UI платформы документации**, не пользовательский контент. | См. **`engine/docs-brand/README.md`**. |
| **`engine/public/`** (стандарт Vite) | Статика движка: иконки, архив **`previous_versions/`**, связь **`content/`** (см. выше). | Не трогать без нужды: конвенция фреймворка. |
| **`archive/`** | Обычно однозначно: **история миграций**, не актуальный код. | Можно оставить; при желании — `migration-archive/` (косметика). |
| **`brief.md` / `CONTEXT.md`** (файлы, не папки) | Оба в корне; новичок может не знать порядок чтения. | Не переименовывать без нужды; в **`README.md`** уже есть навигация по ролям. |

Итого: после выделения **`engine-docs/`** самые «опасные» по смыслу остаются **`0_docs`** и **`references`** внутри контента — их трогают только при осознанной миграции путей, а не «для красоты».
